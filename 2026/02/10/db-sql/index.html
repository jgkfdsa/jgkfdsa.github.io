执行流程

1. 连接与认证（Connection & Authentication）

  • 客户端通过网络（如 TCP/IP）连接到数据库服务器。
  • 数据库验证用户名、密码、权限等。
  • 建立会话（Session），分配资源（如内存、线程）。

✅ 此阶段发生在 SQL 执行前，属于连接层。

2. SQL 解析（Parsing）

数据库接收到 SELECT 语句后，首先进行语法和语义分析：

词法分析（Lexical Analysis）

  • 将 SQL 字符串拆分为“词元”（tokens），如 SELECT、*、FROM、users 等。

语法分析（Syntax Parsing）

  • 检查 SQL 是否符合语法规则（如是否缺少 FROM）。
  • 构建抽象语法树（AST, Abstract Syntax Tree）。

语义分析（Semantic Analysis）

  • 检查表是否存在、列是否合法、用户是否有权限访问。
  • 解析别名、函数、表达式等。

❌ 如果出错（如表不存在），直接返回错误，不继续执行。

3. 查询重写与优化（Query Rewriting & Optimization）

查询重写（Optional）

  • 数据库可能对 SQL 进行等价变换，例如：
    • 视图展开
    • 子查询转 JOIN
    • 常量折叠（如 WHERE 1=1 AND name='Alice' → WHERE name='Alice'）

查询优化（Query Optimization）

  • 生成多个可能的执行计划（Execution Plan）。
  • 基于统计信息（如表行数、索引分布、数据选择性）估算每个计划的成本（Cost）。
  • 选择成本最低的执行计划。

📌 优化器决定：是否用索引？用哪个索引？JOIN 顺序？是否全表扫描？

5. 执行引擎运行（Execution）

数据库的执行引擎按计划一步步操作：

  • 访问存储引擎（如 InnoDB、MyISAM、Heap）
  • 读取数据页（从磁盘或缓冲池/Buffer Pool）
  • 应用过滤条件（WHERE）
  • 排序（ORDER BY） → 可能用内存或临时文件
  • 分组聚合（GROUP BY + 聚合函数）
  • 去重（DISTINCT）
  • 连接（JOIN） → 嵌套循环、哈希连接、排序合并等
  • 限制结果数量（LIMIT）

💡 数据通常以“行流”（row-by-row 或 chunk）方式处理，而非一次性加载全部。

6. 结果返回（Result Delivery）

  • 将最终结果集格式化（如二进制协议、JSON、表格）。
  • 通过网络发送给客户端。
  • 释放本次查询占用的临时资源（如排序内存、临时表）。
