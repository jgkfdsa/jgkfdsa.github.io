基本介绍

  • iter.Seq2 是 Go 1.23 中引入的标准库新特性，用于支持泛型迭代器（iterator）模式。
  • 它是对 Go 中“生成器”或“惰性序列”模式的一种官方、高效且内存友好的实现方式，可以看作是对传统 channel + goroutine 模拟 yield 的一种更轻量、无并发开销的替代方案。

  • 启动 goroutine 有调度开销；
  • 要管理 channel 生命周期；

功能

  • 它表示一个双值迭代器（例如 map 的 key-value 对、索引-元素对等）。
  • 迭代器内部在每次“产出”一对值时，调用 yield(k, v)；
  • 如果 yield 返回 false，迭代器应提前终止（支持 break 语义）。

对比

特性 iter.Seq2 channel + goroutine
是否启动 goroutine ❌ 否 ✅ 是
内存分配 通常为 0（可内联） 至少分配 channel 和 goroutine 栈
支持 break / return ✅ 原生支持 需额外信号（如 context）
并发安全 ❌ 单线程执行 ✅ 天然跨 goroutine
适用场景 本地惰性序列、转换、过滤 跨 goroutine 流式处理

💡 建议：如果不需要并发，优先使用 iter.Seq2；如果涉及 I/O、并行生产消费，则仍用 channel。

核心思想

由调用者提供一个 yield(k, v) 回调，迭代器在需要“产出”一对值时调用它；若 yield 返回 false，则提前终止。
