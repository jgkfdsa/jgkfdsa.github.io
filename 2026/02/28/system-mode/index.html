mode

基本概念

用户态（User Mode）

  • 用户程序运行在用户态。
  • 权限较低：不能直接访问硬件资源（如内存、磁盘、网卡等），也不能执行某些敏感的CPU指令（如修改页表、关闭中断等）。
  • 目的是防止应用程序误操作或恶意破坏系统。

内核态（Kernel Mode）

  • 操作系统内核运行在内核态。
  • 权限最高：可以执行任何CPU指令，访问所有内存地址和硬件设备。
  • 负责管理硬件资源、调度进程、处理中断、提供系统调用接口等核心功能。

优点

  • 安全性：如果所有程序都能随意访问硬件或修改内核数据结构，一个出错或恶意的程序就可能崩溃整个系统。
  • 稳定性：将不可靠的用户程序隔离在受限环境中，即使程序崩溃也不会影响内核和其他进程。
  • 抽象与统一接口：内核通过系统调用向用户提供统一、安全的硬件访问接口，隐藏底层复杂性。

对比

特性 用户态 内核态
权限级别 低（如 x86 的 Ring 3） 高（如 x86 的 Ring 0）
可执行指令 普通指令 所有指令（包括特权指令）
内存访问 受限（只能访问自己的虚拟地址空间） 可访问全部物理/虚拟内存
硬件访问 不可直接访问 可直接控制硬件
出错影响 通常只影响当前进程 可能导致系统崩溃（Kernel Panic）
运行代码 应用程序 操作系统内核

虚拟地址

基本概念

虚拟地址（Virtual Address, VA）

  • 进程看到的地址空间（如 0x400000 ～ 0x7fffffffffff 在 x86-64 上）。
  • 对进程透明，由编译器/链接器生成。

物理地址（Physical Address, PA）

  • 实际 RAM 芯片上的地址。
  • 只有内核和硬件能直接操作。

地址转换

  • CPU 生成虚拟地址 → MMU（Memory Management Unit） 查页表 → 转换为物理地址。
  • 若映射不存在或权限错误 → 触发缺页异常（Page Fault），由内核处理。

优点与代价

✅ 优点

  • 安全隔离：进程互不干扰
  • 简化编程：程序员无需管理物理地址
  • 高效共享：代码段、共享库只需一份物理内存
  • 支持大程序：即使物理内存小，也能运行大型应用（靠 swap）
  • 内存保护：防止缓冲区溢出等攻击（配合 NX bit）

⚠️ 代价

  • 地址转换开销：虽有 TLB 缓解，但仍存在
  • 内存碎片：虚拟地址连续 ≠ 物理连续
  • 复杂性：页表管理、缺页处理、TLB 刷新等增加内核复杂度
  • 性能陷阱：频繁缺页、TLB miss、swap I/O 会显著降低性能

关键功能实现

缺页异常（Page Fault）处理

  1. 当访问未映射或未加载的虚拟页时
  2. CPU 触发 page fault 异常（向量 14）
  3. 内核 do_page_fault() 分析原因：
    • 合法访问但未加载 → 分配物理页，从磁盘/文件/零页加载内容（按需分页）
    • 写时复制（COW） → 父子进程 fork 后共享页，首次写触发 COW
    • 非法访问（如空指针） → 发送 SIGSEGV
  4. 建立页表映射，恢复执行

内存映射（mmap）

  • 将文件或设备映射到进程地址空间（如共享库 .so 文件）。
  • 访问映射区域 → 触发缺页 → 从文件读取内容到内存。
  • 支持匿名映射（无文件 backing，用于堆、栈）。

交换（Swapping）

  • 当物理内存不足时，将不活跃的页写入swap 分区/文件。
  • 下次访问时再换入（Swap-in）。
  • Linux 使用LRU（最近最少使用） 等算法选择换出页。

写时复制（Copy-on-Write, COW）

  • fork() 时子进程不复制父进程内存，而是共享页，并标记为只读。
  • 任一方尝试写 → 触发 page fault → 内核分配新页并复制内容。
  • 极大优化了进程创建性能。
