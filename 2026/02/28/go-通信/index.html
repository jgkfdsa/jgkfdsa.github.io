  • 随机选择一个就绪的 case 执行（避免饥饿）。
  • default 子句使 select 非阻塞。
  • 可用于实现超时、心跳、退出信号等。

cond

底层结构

  1. 调用者必须已持有 L 锁。
  2. Wait() 内部：将当前 goroutine 加入等待队列，原子地释放 L 锁，挂起当前 goroutine
  3. 当被 Signal()/Broadcast() 唤醒后：自动重新获取 L 锁，返回（此时已持有锁）

使用场景

当条件不满足时，goroutine 挂起等待；当其他 goroutine 修改了共享状态并认为条件可能满足时，唤醒等待者去重新检查。
